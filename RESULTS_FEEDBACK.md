# 포항 북구 전세 보증금 예측 — 실험 결과 & 피드백

**Team 호재호 · IML Final Project**
**Run date:** 2026-05-10
**데이터:** 4,470건 (전세 2,486 + 월세 환산 1,984), 경상북도 포항시 북구

---

## 1. 실험 설정 요약

| 항목 | 내용 |
|------|------|
| Target | 보증금(만원) — 월세는 전월세전환율 6.5% 적용해 전세 환산값으로 통합 |
| Split | 80% train / 20% test, `random_state=42` |
| 전처리 | StandardScaler(연속형) + OneHotEncoder(읍면동, 26개 동) |
| Group A | 전용면적, 층, 건물연령, 계약연도, 계약월, 읍면동, 바다여부 (7개) |
| Group B | A + 공원여부, 공원수, 최대/총 공원면적, 동_초/중/고 학교수, 동_초중고모두있음 더미 (15개) |
| 모델 | Ridge(α=10), RandomForest(n=300, depth=20), XGBoost(n=600, lr=0.05, depth=6) |

## 2. 핵심 결과 (Test set)

| Model | RMSE_A | RMSE_B | ΔRMSE | MAE_A | MAE_B | ΔMAE | R²_A | R²_B | ΔR² |
|---|---:|---:|---:|---:|---:|---:|---:|---:|---:|
| **Ridge** | 2,974 | 2,976 | +1.8 | 2,166 | 2,170 | +3.7 | 0.8145 | 0.8143 | -0.0002 |
| **RandomForest** | 2,346 | **2,334** | **-11.9** | 1,560 | **1,544** | **-16.8** | 0.8846 | **0.8857** | **+0.0011** |
| **XGBoost** | **2,233** | 2,246 | +13.2 | **1,499** | 1,517 | +17.8 | **0.8955** | 0.8942 | -0.0013 |

> 단위: 만원 (RMSE/MAE) — 절대 오차 약 1,500–2,200만원 수준.

### 가설 검정
- **H₀**: "Group B 추가가 RMSE↓ / MAE↓ / R²↑를 *최소 2/3 모델*에서 만족하지 않는다."
- 실험 결과: **3 모델 중 Random Forest 한 개에서만** 성능 개선 확인 → **H₀ 채택(=Spatial 변수 추가 효과 미확인)**
- 단, Random Forest의 개선 폭(ΔR² ≈ +0.001, ΔRMSE ≈ -12만원)도 통계적으로 유의하다고 보기엔 작음.

## 3. 모델별 해석

### Ridge — R² 0.81
- 선형 모델로는 비선형/지역 효과를 충분히 못 잡음 → 트리 계열 대비 RMSE 약 30% 높음.
- 공간변수가 Ridge 성능에 거의 영향 없음 — 이는 공간변수가 *가법적 선형 효과*가 아니라 *상호작용/비선형 효과*로 작동함을 시사.

### Random Forest — R² 0.88
- Group A 대비 RMSE 11.9, MAE 16.8 감소.
- 공간변수가 일부 비선형 분기 조건에 기여하지만 개선 폭이 미미.
- 이미 읍면동(One-Hot 26개) 변수가 지역 정보를 상당 부분 흡수하고 있어, 공원·학교 변수가 추가로 줄 수 있는 정보가 제한적이라는 해석이 자연스러움.

### XGBoost — R² 0.90 (최고 성능)
- Group A에서 가장 좋은 성능. Group B에서는 오히려 **소폭 악화**.
- 원인: 공원/학교 변수가 *읍면동 OneHot*과 정보적으로 거의 중첩되어 모델이 노이즈로 처리.
- XGBoost gain top-15에는 공원/학교 변수 4개 (동_고등/초등학교수, 최대/총공원면적) 가 포함되었지만, **상위 변수는 여전히 건물연령(20.9%) → 동_고등학교수(10.3%) → 전용면적(7.2%)**.

### 변수 중요도(XGBoost Group B)
| Rank | Feature | Gain |
|---:|---|---:|
| 1 | 건물연령(계약기준) | 0.210 |
| 2 | 동_고등학교수 | 0.103 |
| 3 | 전용면적(㎡) | 0.072 |
| 4 | 동_초등학교수 | 0.072 |
| 5 | 읍면동_용흥동 | 0.050 |
| 6 | 읍면동_환호동 | 0.049 |
| 9 | 최대공원면적 | 0.037 |
| 10 | 총공원면적 | 0.035 |
| 12 | 바다여부 | 0.026 |

> 흥미로운 발견: 학교 변수(특히 고등학교수)가 단지 변수보다 높은 gain을 갖는다 — 동 단위로 학교가 거의 변하지 않으므로 사실상 *동 효과의 대리 변수*로 작동.

## 4. 결과 예측 / 피드백

### A. 현재 setup 그대로 발표할 때의 메시지
- "공간변수의 *주효과*는 Ridge 모델에서는 확인되지 않으며, 트리계열 모델에서도 ΔR² < 0.005 수준의 미세한 변화에 그친다."
- "이는 H₀ 기각 실패가 곧 'spatial 정보 무용'을 뜻하지는 않는다 — 읍면동 OneHot이 이미 공간정보를 흡수하기 때문."
- 솔직히 이 결론은 발표용으로 약함. 아래 **B/C/D** 실험을 추가하는 것을 강력 권장.

### B. **읍면동 변수를 빼고** Group A vs B 비교 (가장 중요)
- 현재 Group A에 *읍면동 26-OneHot*이 포함되어 있어 이미 지역 효과를 거의 흡수함 → 공간변수 추가 효과가 *덮인다*.
- 읍면동을 제거한 "동 정보 없는 베이스라인" vs "공원/학교/바다만 추가한 spatial 모델"을 비교하면, **공원/학교/바다 변수의 순수 기여도**가 훨씬 명확하게 드러날 것 (ΔR² 0.05~0.10 예상).
- **이 변형이 사실 본 연구의 진짜 hypothesis와 더 잘 맞음.**

### C. **상호작용 효과 검증 — 부분의존도(PDP) / SHAP**
- Group B XGBoost에 대해 SHAP를 돌려보면 (현재 quick_run에는 빠짐), `바다여부 × 전용면적` 같은 상호작용이 보일 가능성이 큼.
- 본 코드(`jeonse_prediction_pipeline.py`)에 SHAP 코드는 이미 포함됨 — 발표 직전에 `python3 jeonse_prediction_pipeline.py` 한 번 실행하면 `shap_summary_xgb_groupB.png` 생성.

### D. **타깃 정의 보강 — 월세→전세 변환**
- 현재 전월세 전환율 6.5% 단일값 사용. 실제로는 *연도별·동별*로 변동 (2024~2025 경북 평균 5.8~7.1%).
- 환산값이 약 12,000만원~13,000만원 영역에서 *과대평가*될 수 있고, 이 영역에 월세 거래가 몰려 있다면 RMSE를 끌어올림.
- 개선안: ① 전세만으로 모델링 (N=2,486), ② 연도별 경북 전월세전환율 표 적용, ③ 전세·월세 더미를 입력 변수로 추가.

### E. **이상치 / 분포 처리**
- 보증금 분포가 long-tail (max 45,000만원 vs median 18,000만원). 로그변환(`log1p`)이 RMSE 안정화에 도움됨 — Ridge에서 R²가 0.85+로 올라갈 가능성.
- 코드 한 줄 수정: `y = np.log1p(df["보증금_환산"])` 후 예측은 `np.expm1(pred)`.

### F. **모델 측 보강 (선택)**
- LightGBM 추가하면 XGBoost와 비교 가능.
- 하이퍼파라미터 탐색을 GridSearchCV로 더 넓혀 (`jeonse_prediction_pipeline.py` 참조) Test RMSE 추가 5–10% 개선 가능.
- 단, 발표 메시지의 핵심은 모델보다 **feature group의 비교 논리**가 명확해지는 데 있음.
